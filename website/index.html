Postmortem of development process

What would we do differently, if we had the chance to start over?

If we had a chance to start over, we would develop our front end such that application view can fit in properly in all android devices. Currently, view dimensions are only compatible with Nexus 7.

Brilliant design decision

During iteration 1, we were expected to test logic layer using fake database of recipes. While discussing the implementation, one of our team members, Yixi Wu came up with an amazing idea of storing recipe data in .csv file. The reason of that is to make things easier while implementing actual database. Because, SQL server creates table directly by reading .csv file. As a result, we saved a lot of our time writing insert queries.

What took the most time? The least? Any surprises?

The things that took the most time was Database and GUI design. For Database, we need to have some default data that can populate our database(even the stub database), so that the app wouldn't look so dull. Creating and adding the Recipes to the csv file for our app really took a long time. And setting up the database in android studio is time consuming as well. Android studio's dependency makes it you have to be an Activity class to have access to assets folder, which creates difficulty for both testing and developing. Another things is that, a changes to the assets folder will not be copied over to where the actual database is stored, you have to un-install the app to make the changes show, which gives us unexpected bugs and took a really long time for us to figure out. But still, setting up the CSV file for the database is a good decision. It helped my reduce my time to add data to the real database since converting the data from CSV file to a insert query script is easy.For GUI design, most of us don't have enough experience with Android GUI Design, so we have to learn a lot of new things to develop the GUI. It took us a really long time to find suitable component for our App. And leraning takes time, debugging takes time, so the process for GUI is just really time-consuming. The things that took the least time is creating and designing object for the app. We were thinking that "Object belongs to a Data/Persitence Layer, that sounds fancy so it must take us some time to design and implement it!". But in reality, it is not that hard, object is... just object in Java. Figuring out what field should an Recipe Object to have is not that hard. The surprises part is that, creating an Interface is easier than it looks. "Dependency Injection" sounds really fancy, but in the end, It was just using Interface and taking parameter as input. It is amazing that how simple a concept is, that can make our software development much easier in the future, if we can do it right.
